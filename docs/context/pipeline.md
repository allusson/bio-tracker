The src/pipeline/ directory contains the core algorithm stages of the localization pipeline. Each module takes the output of the previous stage and adds one layer of inference. All stages are controlled by config.py at the repo root.

src/pipeline/wrangler.py implements Stage 1: converting the raw long-format scan table into the wide-format RSSI vector table that the clustering stage needs. It exports a single function wrangle(df) that accepts a long-format DataFrame (gateway_id, timestamp, device_id, rssi) and returns a pivoted DataFrame indexed by (device_id, time_window) with hub IDs as columns.

wrangle() performs four steps. First, it floors each timestamp to the nearest WINDOW_MINUTES boundary by calling floor_timestamp() from utils/helpers.py and storing the result in a new time_window column. Second, it pivots the long table to wide format using pd.pivot_table() with index=(device_id, time_window), columns=gateway_id, values=rssi, and aggfunc="last". NaN appears wherever a hub did not see the device in that window. Third, it filters out rows (device-window pairs) where the number of non-NaN hub values is below MIN_HUBS_PER_WINDOW; a vector with only one hub reading lacks directional information for localization. Fourth, it prints a diagnostic summary: total windows before and after filtering, percentage with two or more hubs, and an ASCII bar chart of hub-count distribution.

The aggfunc="last" behavior is critical and non-obvious: pd.pivot_table() keeps the last row in current DataFrame order, not the row with the latest timestamp. It does not sort internally. The preprocessor sorts by (device_id, timestamp) before data reaches wrangle(), so ordering is correct for preprocessed real data. If an unsorted DataFrame is passed directly, the "last" reading may not be chronologically most recent.

The 20-minute window is required for real data and is not configurable downward. BioHub nodes have independent fixed clock offsets reaching up to ~9.5 minutes across the hub population. This is a reporting artifact — hub report timestamps cannot be interpreted as the moment a device was exclusively in one hub's range. A 10-minute window splits hub reports for the same logical scan cycle across different buckets, fragmenting RSSI vectors. The empirical evidence is in notebooks/02_multi_hub_visibility.ipynb.

WINDOW_TOLERANCE_MINUTES (default 2) is defined in config.py but is not yet used inside wrangler.py.

The output is a MultiIndex DataFrame. Rows are indexed by (device_id, time_window). Columns are hub IDs. Values are RSSI in dBm. NaN means that hub did not see the device in that window. Only rows with at least MIN_HUBS_PER_WINDOW non-NaN values are included.

src/pipeline/clustering.py is a stub. When implemented, it will accept the wide-format RSSI vector DataFrame and return it with an added zone column. NaN values must be imputed or masked before vectors are passed to scikit-learn estimators, which do not accept NaN inputs. Three methods are planned, controlled by config.CLUSTERING_METHOD: kmeans (fast baseline, hard zone assignments, requires N_ZONES), gmm (soft probabilistic assignments, better upstream input for the HMM, also requires N_ZONES), and dbscan (density-based, automatically determines number of zones, marks noise observations as zone -1).

src/pipeline/hmm.py is a stub. When implemented, it will accept the DataFrame with the zone column from clustering.py and return it with an added smoothed_zone column. Hidden states are the zones discovered by clustering. The transition matrix will be estimated from empirical zone-to-zone transition frequencies in the data. Emissions will be modeled from RSSI vectors or GMM cluster probabilities. Viterbi decoding via hmmlearn will recover the most probable zone sequence for each device over time.

Configuration parameters from config.py relevant to this directory: WINDOW_MINUTES (default 20) controls bucket size in wrangler. MIN_HUBS_PER_WINDOW (default 2) controls the filter threshold in wrangler. CLUSTERING_METHOD (default "kmeans") controls which algorithm clustering.py will use. N_ZONES (default 4) sets the number of clusters for k-means and GMM. HMM_N_ITER (default 100) sets training iterations for the HMM.

Dependencies: pandas for pivoting, filtering, and MultiIndex handling in wrangler.py; scikit-learn for k-means and GMM in clustering.py (planned); hmmlearn for HMM and Viterbi in hmm.py (planned).
